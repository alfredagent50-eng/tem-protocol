# Open Source Strategy

Open source is useful for `tem-protocol` if we treat it as a trust and adoption advantage, not as charity.

## Why open source helps

- Payments and calendar access require trust.
- Protocol primitives benefit from public review.
- Developers may build integrations if the model is open.
- A transparent protocol can become bigger than one hosted SaaS app.

## What can stay commercial later

- Hosted convenience
- Premium integrations
- Managed payment routing
- Team/organization features
- Advanced analytics
- Native mobile polish

## Recommended stance

Start public with an MIT license while the project is young. Avoid publishing secrets or business-sensitive execution details. Keep the protocol, models, and basic app open; monetize hosting and premium workflows later.
