# Roadmap

## Phase 0 — Shape the protocol

- Define booking lifecycle
- Define payment lifecycle
- Define refund/cancellation semantics
- Decide first payment rail
- Decide MVP calendar integration depth

## Phase 1 — MVP web app

- Host profile and booking page
- Slot rules and minimum payment
- Guest booking flow
- Payment proof abstraction
- Host approval/rejection
- Calendar invite generation

## Phase 2 — Real payment rails

- Lightning invoices or LNURL
- Stablecoin payment option
- Payment status webhooks/listeners
- Refund handling

## Phase 3 — Protocol/API

- Public API
- Webhooks
- Embeddable booking widget
- Self-hosting docs

## Phase 4 — Mobile/iOS

- Mobile-first booking flow
- Push notifications for offers
- Quick accept/reject
